# Equality Ombudsman Web Form Security Case

## Case type

IMY decision / authority decision

## Jurisdiction

Sweden

## Year

2025

## Authority or court

IMY, Integritetsskyddsmyndigheten

## Topic

GDPR, personal data breach, web form, security measures, potentially sensitive personal data, personal data processor, website analytics, administrative fine.

## Short summary

IMY issued an administrative fine of SEK 100,000 against the Equality Ombudsman, Diskrimineringsombudsmannen, after a personal data incident connected to a web form used for tips and complaints about discrimination.

The case concerned a form where individuals could submit information that may be sensitive or highly personal, depending on the content of the tip or complaint.

The Equality Ombudsman had taken a security measure intended to prevent personal data from the form from being included in usage analyses of the website.

However, that security measure did not work as intended.

As a result, some information from the form was inadvertently disclosed to the personal data processor that had been hired to carry out the website analyses.

According to IMY, approximately 500 tips and complaints were estimated to have been affected.

The case is important because it shows that security measures must be effective in practice, not only planned or documented.

It also shows that ordinary web forms and analytics tools can create GDPR risk when personal data is collected, processed, or shared in ways that were not intended.

The incident was not a classic cyberattack, ransomware case, or malicious intrusion.

Instead, the problem came from a failed control in a normal digital service, which makes the case useful for understanding everyday GDPR security risks.

## What happened

The Equality Ombudsman used a web form to collect tips and complaints about discrimination.

The form could contain personal data, and depending on what a person wrote, the information could also include sensitive or potentially sensitive details.

The incident was reported to IMY in the fall of 2021.

During IMY's supervision, it emerged that the Equality Ombudsman had taken a security measure intended to protect the personal data collected through the web form.

The purpose of that measure was to make sure that the form data would not be included in usage analyses of the website.

However, the security measure did not work as intended.

As a result, some data from the form was inadvertently disclosed to the personal data processor hired to conduct the analyses.

According to IMY, approximately 500 tips and complaints were estimated to have been affected.

The incident shows how a privacy or security problem can arise through a normal website function when the technical setup does not properly separate form data from analytics processing.

## Legal issue

The main legal issue was whether the Equality Ombudsman had taken sufficiently effective security measures to protect personal data collected through the web form.

The case connects to GDPR security, personal data breach assessment, sensitive personal data, processor relationships, web forms, analytics, and technical and organisational measures.

It also raises the question of how organizations should verify that a security measure actually works as intended after it has been implemented.

## Decision or outcome

IMY concluded that the Equality Ombudsman did not take sufficiently effective security measures.

IMY issued an administrative fine of SEK 100,000.

The decision shows that it is not enough to have a security measure in theory.

The organization must ensure that the measure actually works for the processing risk.

The case also shows that a disclosure to a processor can still be a serious problem if the processor receives personal data that it should not have received for that purpose.

## Swedish decision or outcome

Beslutet visar att det inte räcker att ha en säkerhetsåtgärd i teorin.

Organisationen måste säkerställa att åtgärden faktiskt fungerar i förhållande till risken med behandlingen.

Fallet visar också att ett röjande till ett personuppgiftsbiträde fortfarande kan vara ett allvarligt problem om biträdet tar emot personuppgifter som det inte skulle ha tagit emot för det aktuella ändamålet.

## Why it matters for CyberLex

This case is important because it gives a clear example of GDPR risk connected to ordinary web forms and website analytics.

The case shows that personal data incidents do not always involve hacking, ransomware, or malicious actors.

A personal data breach can also happen when:

* web forms are configured incorrectly
* analytics tools receive data they should not receive
* security measures fail
* sensitive or potentially sensitive information is processed through a form
* processor arrangements and technical controls are not effective enough
* implemented controls are not properly tested
* form data is accidentally included in website usage analysis
* the organization does not notice that a technical safeguard is failing

The case helps explain that incident prevention and GDPR compliance require both planning and verification.

It also shows that web forms should be treated as high-risk when users may submit sensitive, personal, or complaint-related information.

## Learning note

This case shows that a security measure is not enough just because it exists.

Organisations must verify that the measure actually works for the processing risk, especially when web forms may collect sensitive or potentially sensitive information.

The case also shows that analytics and usage-analysis tools should be reviewed carefully before they are connected to forms, complaint systems, contact pages, or other areas where users may submit personal data.

A good control should not only be designed. It should also be tested, monitored, documented, and reviewed when the website or analytics setup changes.

Fallet visar också att analysverktyg och användningsanalys bör granskas noggrant innan de kopplas till formulär, klagomålssystem, kontaktsidor eller andra delar där användare kan lämna personuppgifter.

En bra kontroll ska inte bara vara utformad. Den bör också testas, övervakas, dokumenteras och granskas när webbplatsen eller analyslösningen ändras.
